# PaperBridge

PaperBridge 是我做的一个文献阅读翻译助手。它的目标很简单：让我在阅读英文数学文献时，不需要频繁复制、切换网页、粘贴翻译，而是可以直接在 PDF 原页上选中一句话或一段话，马上在右侧看到中文译文。

我自己是数学专业的学生。读论文、教材和讲义时，很多困难并不是数学本身，而是语言带来的额外负担。尤其是遇到长句、证明段落、定义和定理陈述时，我希望能保留原始 PDF 的版面、公式和上下文，同时快速得到一份中文辅助理解。因此我做了这个双栏工具：左边读原文，右边看译文。

## 在线试用

这个项目可以部署成静态网页。部署后，使用者只需要打开链接，就可以导入自己的 PDF 文件并开始阅读。

需要说明的是：PDF 文件是在浏览器本地打开和渲染的，不会因为“打开 PDF”这个动作自动上传到服务器。只有当你使用在线翻译引擎时，被选中的那一小段文字才会发送给对应的翻译服务。

如果我已经把它部署到了 GitHub Pages 或其他网页平台，最简单的使用方式就是直接打开网页链接。这个方式不需要安装任何软件，也不需要会命令行。

## 第一次使用指南

如果你只是想试用在线版本：

1. 打开我分享的 PaperBridge 网页链接。
2. 点击右上角的“打开文献”。
3. 选择你电脑里的 PDF 文件。
4. 在左侧 PDF 原文中选中英文句子或段落。
5. 在右侧查看中文译文。

如果你想把项目下载到自己电脑上运行，可以按照下面的步骤来。

### Windows 用户

1. 打开这个项目的 GitHub 页面。
2. 点击绿色的 `Code` 按钮。
3. 选择 `Download ZIP`。
4. 把下载好的压缩包解压到一个你能找到的位置，例如桌面。
5. 安装 Node.js：到 Node.js 官网下载并安装 LTS 版本。
6. 打开解压后的 `PaperBridge` 文件夹。
7. 在文件夹空白处按住 `Shift`，点击鼠标右键，选择“在终端中打开”或“在 PowerShell 中打开”。
8. 第一次运行时，输入：

```powershell
npm install
```

9. 安装完成后，输入：

```powershell
npm run dev
```

10. 终端里会出现一个本地网址，通常是：

```text
http://127.0.0.1:5176/
```

11. 用浏览器打开这个网址，就能看到 PaperBridge 界面。

以后再次使用时，一般只需要打开 `PaperBridge` 文件夹，在终端里运行：

```powershell
npm run dev
```

### macOS 用户

1. 打开这个项目的 GitHub 页面。
2. 点击绿色的 `Code` 按钮。
3. 选择 `Download ZIP`。
4. 解压下载好的文件。
5. 安装 Node.js LTS 版本。
6. 打开“终端”应用。
7. 输入 `cd `，后面拖入解压后的 `PaperBridge` 文件夹，然后按回车。
8. 第一次运行时输入：

```bash
npm install
```

9. 安装完成后输入：

```bash
npm run dev
```

10. 用浏览器打开终端里显示的本地网址，通常是：

```text
http://127.0.0.1:5176/
```

### 常见问题

- 如果提示 `npm 不是内部或外部命令`，通常说明 Node.js 没有安装好，或者安装后还没有重新打开终端。
- 如果 `npm install` 比较慢，可以换一个网络环境后再试。
- 如果浏览器打不开 `http://127.0.0.1:5176/`，请确认运行 `npm run dev` 的终端窗口没有关闭。
- 如果端口被占用，终端可能会显示另一个网址，直接打开终端里实际显示的网址即可。

## 功能

- 在浏览器中打开 PDF、TXT、Markdown 文献。
- PDF 会以原始页面形式显示，尽量保留排版、公式、图片和上下文。
- 左侧 PDF 阅读区可以独立滚动，右侧翻译区不会跟着移动。
- 支持输入页码并直接跳转到指定页面。
- 在左侧选中英文句子或段落后，右侧自动显示中文译文。
- 支持翻译历史和一键复制译文。
- 支持三种翻译来源：公共翻译、自定义端点、本地术语 fallback。

## 如何使用

1. 打开 PaperBridge 页面。
2. 点击右上角的“打开文献”。
3. 选择本地 PDF、TXT 或 Markdown 文件。
4. 如果是 PDF，可以在左侧直接阅读原始页面，也可以在顶部输入页码跳转。
5. 用鼠标选中一句或一段英文。
6. 右侧会自动显示当前选区和中文译文。

如果你只是想快速体验，可以使用默认的“公共翻译”。如果你对翻译质量和隐私有更高要求，可以切换到“自定义端点”，接入你自己的翻译服务。

## 翻译引擎说明

当前默认的“公共翻译”使用 MyMemory Translated API：

```text
https://api.mymemory.translated.net/get
```

这个默认服务适合原型演示和轻量试用，但我不认为它适合严肃、大规模的学术阅读场景。更好的方式是接入自己的翻译端点，例如 OpenAI、DeepL、学校内部服务或本地大模型。

“本地术语”模式不会联网，只做非常粗略的数学术语替换。它主要是网络失败时的 fallback，不应该被当成正式翻译。

## 使用 OpenAI 翻译

不要把 OpenAI API key 写进前端网页。浏览器里的 key 很容易被别人看到和滥用。

我在项目里放了一个本地 OpenAI 翻译代理。你可以在自己的电脑或服务器上运行它，再让 PaperBridge 的“自定义端点”去调用这个代理。

PowerShell 示例：

```powershell
$env:OPENAI_API_KEY="sk-..."
npm run translate:openai
```

然后在 PaperBridge 页面右上角打开设置，选择“自定义端点”，填写：

```text
http://127.0.0.1:8787/translate
```

默认模型是：

```text
gpt-5.4-mini
```

如果想换模型，可以这样设置：

```powershell
$env:OPENAI_TRANSLATION_MODEL="gpt-5.5"
npm run translate:openai
```

如果以后要给很多人一起使用 OpenAI 翻译，我会建议把这个代理部署到服务器，并加上登录、额度限制和访问控制。否则 API key 和费用都会有风险。

## 部署到 GitHub Pages

我已经在项目里加入了 GitHub Pages 的自动部署 workflow：

```text
.github/workflows/deploy-pages.yml
```

推荐步骤：

1. 在 GitHub 新建一个仓库。
2. 把这个项目推送到仓库的 `main` 或 `master` 分支。
3. 进入仓库的 `Settings` -> `Pages`。
4. 在 `Build and deployment` 里选择 `GitHub Actions`。
5. 等待 Actions 自动构建和部署。
6. 部署完成后，GitHub 会给出一个可以公开访问的网页链接。

项目里的 `vite.config.ts` 已经处理了 GitHub Pages 常见的路径问题。普通项目站点会自动使用 `/<仓库名>/` 作为 base path；如果仓库是 `用户名.github.io` 这种用户站点，则使用 `/`。

## PDF 图片和特殊字体

PDF 渲染使用 PDF.js。为了尽量支持更多 PDF，我把 PDF.js 的辅助资源放到了：

```text
public/pdfjs
```

其中包括：

- `cmaps`：字符映射资源。
- `standard_fonts`：标准 PDF 字体。
- `wasm`：JPEG2000、JBIG2 等图片解码相关资源。
- `iccs`：颜色配置资源。

如果某个 PDF 的图片仍然显示不出来，可能原因包括：

- 图片并没有嵌入 PDF，而是外部链接。
- PDF 使用了浏览器版 PDF.js 尚不支持的特性。
- PDF 有加密、权限限制或损坏。
- PDF 是扫描版图片，需要 OCR 后才能选中文字和翻译。

## 隐私说明

我希望这个工具尽量尊重使用者的文献隐私：

- 打开 PDF 文件时，文件内容在浏览器本地处理。
- 默认不会把整篇 PDF 上传到服务器。
- 使用公共翻译时，只有被选中的文本会发送到 MyMemory。
- 使用自定义端点时，选中文本会发送到你填写的服务地址。
- 如果你接入 OpenAI、DeepL 或其他服务，需要遵守对应服务的隐私和使用条款。
